# Data files

All CSV schemas are `utf-8`, comma-separated, header row on line 1.

## Phase 1: synthetic training data

Generated by `benchmark/` under `REBUILD_SYNTHETIC=1 make phase1`. The `_full`
file is stratified-split (70 / 15 / 15, `config.SEED = 42`) into train / val /
test. Filenames include the generation timestamp and a content hash so the
committed artefact is unambiguously identifiable.

| File | Rows | Purpose |
| --- | --- | --- |
| `synthetic_final_20260304_205519_c9e4df6d_full.csv` | 4186 | Full pool before splitting. |
| `synthetic_final_20260304_205519_c9e4df6d_train.csv` | 2927 | Router training set. |
| `synthetic_final_20260304_205519_c9e4df6d_val.csv` | 625 | Validation fold (Platt scaling, τ selection). |
| `synthetic_final_20260304_205519_c9e4df6d_test.csv` | 634 | In-distribution test fold. |
| `synthetic_final_20260304_205519_c9e4df6d_metadata.json` | — | Generation config, seed, per-category and per-subcategory counts, dedup stages. |

Columns: `prompt`, `category`, `subcategory`, `style`, `noise`, `params`,
`top_category`.

## Phase 3: evaluation benchmark

Built by `evaluation/dataset_builder.py` from the test fold + full pool under
the polarity-admission rule. This is the 305-pair set the headline
routing-accuracy number is measured on.

| File | Rows | Purpose |
| --- | --- | --- |
| `evaluation_dataset.csv` | 305 | The benchmark: prompt + conversation + ground truth. |
| `evaluation_with_llm.csv` | 305 | Same pairs, extended with cached LLM-only baseline answers, tokens, and latencies. Produced by Phase 3 when `REBUILD_E2E=1`. |

Per-category sizes: SYMBOLIC_TIME 65, SYMBOLIC_COUNT 60, SEMANTIC 60,
SYMBOLIC_METADATA 50, HYBRID 40, UNSUPPORTED 30.

Ground-truth provenance (`ground_truth_source` column): 175 deterministic (all
symbolic pairs, computed by `evaluation/deterministic.py`), 100 judged by
GPT-5.2, 30 rejected (UNSUPPORTED).

## Cross-generator holdout

`holdout_cross_generator.csv` (180 rows) is an out-of-distribution holdout
generated by a separate prompting path (no templates, no forbidden-word list,
no few-shot examples, seed 99). Used to measure generalization gap in Phase 2.
`holdout_metadata.json` records the generation config and confirms zero prompt
overlap with the training set.

## Human-annotation artefacts

`annotations/` holds the materials for the inter-annotator agreement study
reported in thesis Chapter 4.

| File | Purpose |
| --- | --- |
| `ANNOTATION_GUIDELINES.md` | Instructions given to the two human annotators for the 120-item SEMANTIC / HYBRID sanity check. |
| `annotation_study.csv` | Shared annotation pool before labels. |
| `annotation_study_annotator_{1,2}.csv` | Each annotator's returned labels. |
| `annotation_answer_key.csv` | Resolved ground truth after disagreement adjudication. |
| `annotation_disagreements.csv` | Items where the two annotators disagreed, with resolution. |
| `SPOTCHECK_GUIDELINES.md` | Instructions for the smaller semantic-spotcheck study. |
| `semantic_spotcheck_annotator_{1,2}.csv` | Spotcheck labels from each annotator. |
| `semantic_spotcheck_answer_key.csv` | Spotcheck resolved ground truth. |

Reproduce the κ / PABAK / per-category agreement numbers with:

```bash
python -m scripts.analyses.compute_kappa
python -m scripts.analyses.analyze_semantic_spotcheck
```
